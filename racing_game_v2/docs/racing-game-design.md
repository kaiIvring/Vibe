# 赛车游戏 —— 需求与设计文档

## 1. 概述

一个基于 HTML5 Canvas 的简易赛车游戏，全部代码位于单个 `index.html` 文件中，不依赖任何外部库。

## 2. 功能需求

| ID | 需求 | 说明 |
|----|------|------|
| F1 | 玩家控制 | 键盘 ← → 方向键控制车辆左右移动 |
| F2 | 障碍生成 | 障碍车从屏幕顶部随机水平位置出现，匀速下落 |
| F3 | 碰撞检测 | 玩家车与障碍车碰撞 → Game Over |
| F4 | 重新开始 | Game Over 后按 R 键重置游戏 |
| F5 | 计分 | 右上角实时显示分数，存活时间越长分数越高 |
| F6 | 难度递增 | 障碍车下落速度随分数缓慢提升 |

## 3. 技术方案

- **渲染方式**：HTML5 Canvas（全屏铺满）
- **游戏循环**：`requestAnimationFrame`
- **碰撞检测**：矩形包围盒（AABB）
- **状态管理**：游戏状态机（playing → gameover → playing）
- **文件结构**：单个 `index.html`（内嵌 CSS + JavaScript）

## 4. 视觉设计

- **道路**：深灰色背景 + 白色虚线车道分隔线
- **玩家车**：蓝色车身（方块 + 梯形车窗 + 圆形车轮），位于屏幕底部
- **障碍车**：红色/橙色车身（与玩家车相同绘制风格），从顶部下落
- **分数**：右上角白色文字
- **Game Over**：屏幕中央半透明遮罩 + 大号文字提示 + 重新开始指引

## 5. 游戏架构

```
index.html
├── <style>       — 页面样式、Canvas 铺满、分数定位
├── <canvas>      — 游戏主画布
└── <script>      — 游戏逻辑
    ├── 常量配置       — 车辆尺寸、速度、颜色、车道数
    ├── 游戏状态       — playing / gameover
    ├── 玩家对象       — x, y, width, height, speed, draw()
    ├── 障碍物数组     — [{x, y, width, height, speed, draw()}, ...]
    ├── 分数变量       — 帧计数 / 时间累加
    ├── 键盘事件       — keydown（← → R）
    ├── 游戏循环       — update() + draw() + requestAnimationFrame
    ├── 碰撞检测       — rectIntersect(a, b)
    ├── 障碍生成       — 定时/随机间隔生成
    └── 重置逻辑       — 清空障碍、重置位置、重置分数
```

## 6. 核心逻辑

### 6.1 游戏循环

```
function gameLoop() {
    if (state === 'playing') {
        update()    // 更新位置、碰撞检测、生成障碍、计分
    }
    draw()         // 绘制所有内容（playing 和 gameover 状态都绘制）
    requestAnimationFrame(gameLoop)
}
```

### 6.2 碰撞检测

矩形 A 与矩形 B 相交当且仅当：
```
A.x < B.x + B.width  &&
A.x + A.width > B.x  &&
A.y < B.y + B.height &&
A.y + A.height > B.y
```

### 6.3 障碍生成策略

- 每隔一定时间（或帧数）生成一辆障碍车
- 生成间隔随分数增加而缩短
- 障碍车水平位置在道路范围内随机
- 每辆障碍车有独立的下落速度（基准速度 + 分数加成）

### 6.4 难度曲线

- 基准下落速度：2 px/frame
- 每得 100 分速度增加 0.2 px/frame
- 最大速度上限：8 px/frame
- 障碍生成间隔从 60 帧逐渐缩短至 20 帧

## 7. 边界情况

| 场景 | 处理方式 |
|------|----------|
| 窗口缩放 | Canvas 自动 resize 适应窗口 |
| 玩家车超出左右边界 | 限制 x 坐标在道路范围内 |
| 障碍车移出屏幕底部 | 从数组中移除，释放内存 |
| 快速按键 | 每帧只处理一次移动，防止穿透 |
| R 键连按 | 仅在 gameover 状态响应重置 |

## 8. 非功能性需求

- 所有代码在单个 HTML 文件中完成
- 不加载任何外部资源（字体、图片、库）
- 兼容 Chrome / Firefox / Edge 最新版本
- 在普通硬件上保持 60fps
